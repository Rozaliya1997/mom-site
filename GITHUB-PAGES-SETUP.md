# 🚀 Публикация сайта на havizovavenera.com

Персональная пошаговая инструкция: GitHub Pages + Namecheap → твой домен с HTTPS.

**Время:** ~20 минут активных действий + до 1-2 часов ожидания DNS (иногда мгновенно).

> Я уже вписала твой домен `havizovavenera.com` в файл `CNAME`. Этот файл должен лежать в корне репозитория — без него GitHub Pages не подключит твой домен.

---

## ⚠️ Сначала проверь написание домена

Домен у тебя написан как **havizovavenera.com** — через **«h»**, не «hafizova». Видимо, на момент покупки `hafizovavenera.com` был занят.

Это нормально и работать будет — но имей в виду:
- В Google и Яндекс латинская транслитерация фамилии «Хафизова» обычно «Hafizova», и люди по привычке могут вбивать его в адресной строке
- Если позже захочешь сменить домен на правильный — сайт легко перенести, всё то же самое

Можно ничего не менять — `havizovavenera.com` тоже хорошо запоминается. Дальше в инструкции — твой реальный домен.

## Шаг 1. Создать репозиторий на GitHub

1. Зайди на https://github.com → нажми **«+»** в правом верхнем углу → **New repository**
2. Заполни:
   - **Repository name:** `mom-site` (или любое — например `hafizova-buh`)
   - **Description:** `Сайт-визитка бухгалтера` (можно оставить пустым)
   - **⚠️ Public** — обязательно публичный, иначе на бесплатном тарифе Pages не работает
   - ✅ Поставь галочку **Add a README file**
3. Нажми **Create repository**

---

## Шаг 2. Загрузить файлы сайта

1. На странице созданного репозитория нажми **Add file** → **Upload files**
2. Открой папку `site-classic` на своём компьютере
3. **Выдели всё содержимое** (Ctrl+A на Windows / Cmd+A на Mac) — должны выделиться:
   - `index.html`
   - `CNAME` ← важно, без него домен не подключится
   - папка `assets/`
   - папка `blog/`
   - `google-apps-script.gs`
   - `GOOGLE-SHEETS-SETUP.md`
   - `GITHUB-PAGES-SETUP.md`
   - `README.md`
4. Перетащи всё мышкой в окно браузера (туда, куда показывает GitHub)
5. Подожди, пока всё загрузится (зелёные галочки рядом с каждым файлом)
6. Внизу в поле **Commit changes** оставь сообщение по умолчанию или впиши `Initial site upload`
7. Нажми зелёную кнопку **Commit changes**

После этого в репозитории должны быть видны все файлы. Проверь, что папки `assets` и `blog` тоже загрузились (внутри них файлы с картинками и статьями).

---

## Шаг 3. Включить GitHub Pages

1. В репозитории открой вкладку **Settings** (справа сверху)
2. В левом меню найди **Pages**
3. В разделе **Build and deployment** → **Source** выбери **Deploy from a branch**
4. В **Branch** выбери `main` и папку `/ (root)` → нажми **Save**
5. Подожди 1-2 минуты, обнови страницу

Сверху появится надпись:
> ✅ Your site is live at `https://username.github.io/mom-site/`

(вместо `username` будет твой ник на GitHub, вместо `mom-site` — название твоего репозитория)

**Открой эту ссылку** — сайт уже работает! Это временный адрес. Проверь:
- ✅ Главная открывается, видны фото
- ✅ Кликаешь на статью блога → она открывается
- ✅ Кнопки «Записаться» прокручивают к форме

Если что-то не так — лучше починить сейчас, до подключения домена.

## Шаг 4. Настроить DNS у Namecheap

Это самый ответственный шаг. Не торопись, тут важно правильно ввести значения.

### 4.1. Открыть DNS-настройки

1. Зайди в свой аккаунт https://www.namecheap.com
2. Сверху справа: **Account** → **Domain List**
3. Найди в списке `havizovavenera.com` → нажми **Manage** (справа от домена)
4. Перейди на вкладку **Advanced DNS**

### 4.2. Удалить лишние записи

В разделе **Host Records** Namecheap по умолчанию создаёт несколько записей — обычно это:
- `URL Redirect Record` с Host `@` → ведёт на `parkingpage.namecheap.com`
- `CNAME Record` с Host `www` → ведёт на `parkingpage.namecheap.com`

**Удали их все.** Кликни на иконку корзины 🗑️ справа от каждой строки.

### 4.3. Добавить четыре A-записи

Нажми **Add New Record** и добавь по очереди эти четыре строки. После каждой — нажми зелёную галочку справа, чтобы сохранить.

| Type | Host | Value | TTL |
|---|---|---|---|
| **A Record** | `@` | `185.199.108.153` | Automatic |
| **A Record** | `@` | `185.199.109.153` | Automatic |
| **A Record** | `@` | `185.199.110.153` | Automatic |
| **A Record** | `@` | `185.199.111.153` | Automatic |

Это IP-адреса серверов GitHub Pages. Они одинаковые для всех сайтов на GitHub. Копируй прямо отсюда, чтобы не сделать опечатку.

### 4.4. Добавить CNAME для www

Это нужно, чтобы `www.havizovavenera.com` тоже работал и автоматически переадресовывал на `havizovavenera.com`.

| Type | Host | Value | TTL |
|---|---|---|---|
| **CNAME Record** | `www` | `username.github.io` | Automatic |

⚠️ **Замени `username` на свой ник на GitHub** (тот, под которым ты создавала репозиторий — он отображается в адресе твоего репозитория `github.com/username/mom-site`).

Важно:
- Без `https://`
- Без слеша `/` на конце
- Без имени репозитория (только `username.github.io`, не `username.github.io/mom-site`)
- Namecheap может автоматически добавить точку в конце — это нормально

### 4.5. Проверь итог

После всего у тебя в Host Records должно быть **5 записей**:
- 4 строки `A Record` с Host `@` и разными IP-адресами `185.199.108-111.153`
- 1 строка `CNAME Record` с Host `www` и значением `username.github.io`

## Шаг 5. Сказать GitHub, что мы используем havizovavenera.com

> Если ты загрузила файл `CNAME` в репозиторий (как было в Шаге 2) — этот шаг уже частично сделан, GitHub сам подхватил домен из файла. Но всё равно проверь по инструкции ниже.

1. В репозитории на GitHub: **Settings** → **Pages**
2. В разделе **Custom domain**:
   - Если поле уже заполнено `havizovavenera.com` — отлично, GitHub сам подхватил из файла CNAME
   - Если пусто — впиши **`havizovavenera.com`** (без `https://`, без `www`) → нажми **Save**
3. GitHub начнёт проверку DNS. Сначала может писать «**DNS check in progress**» — это нормально

---

## Шаг 6. Подождать обновления DNS

DNS-записи обновляются по миру не моментально. Обычно:
- **5-15 минут** — у большинства провайдеров
- **До 1-2 часов** — в среднем
- **До 24 часов** — гарантированный максимум

**Как проверить, что DNS обновился:**

1. Открой https://dnschecker.org/
2. В поле введи `havizovavenera.com`
3. Тип записи выбери **A**
4. Нажми **Search**
5. На карте мира должны появиться IP: `185.199.108.153`, `109.153`, `110.153`, `111.153`

Когда на большинстве серверов мира видны эти IP — значит DNS дошли.

---

## Шаг 7. Включить HTTPS

Когда GitHub увидит, что DNS настроены правильно, в **Settings → Pages** появится галочка:

☐ **Enforce HTTPS**

**Поставь её.** Это включит бесплатный SSL-сертификат от Let's Encrypt — сайт будет открываться по `https://` с зелёным замочком в браузере.

> Если галочка серая и не активируется — DNS ещё не дошли до серверов GitHub. Подожди час-два и обнови страницу.

## Если что-то пошло не так

### «Сайт не открывается, домен не находит»
- Подожди 1-2 часа после настройки DNS — это самая частая причина
- Проверь на https://dnschecker.org что 4 IP-адреса видны
- Убедись что **все 4** A-записи добавлены и у всех Host = `@` (не пусто, не `havizovavenera.com`, именно `@`)

### «Открывается, но без картинок»
- Скорее всего, в репозитории нет папки `assets/`. Открой репозиторий и проверь
- Если папка есть, но пустая — заново загрузи через **Add file → Upload files**

### «Не открываются статьи блога»
- Аналогично — проверь, что в репозитории есть папка `blog/` с тремя HTML-файлами и `article.css`

### «Custom domain выдаёт ошибку 'not properly configured'»
- DNS ещё не обновились — просто подожди
- Или в записи CNAME для `www` ты вписала неправильный username

### «Не активируется галочка Enforce HTTPS»
- DNS ещё не дошли до серверов GitHub. Подожди час и попробуй снова
- Иногда GitHub-у нужно до 24 часов после первой настройки, чтобы выпустить SSL-сертификат

### «Форма не отправляет заявки»
- Проверь, что в `index.html` заменила `PASTE_YOUR_GOOGLE_APPS_SCRIPT_URL_HERE` на свой URL Apps Script (см. файл `GOOGLE-SHEETS-SETUP.md`)
- Если ещё не настраивала Google Sheets — настрой по той инструкции

### «На havizovavenera.com открывается чужой сайт / parking-страница Namecheap»
- Старые DNS-записи Namecheap не удалены. Вернись к Шагу 4.2 и удали все Redirect/CNAME записи, которые там стояли по умолчанию
